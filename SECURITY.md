# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | ✅ (当前版本)     |
| < 0.1   | ❌                |

## Reporting a Vulnerability

If you discover a security vulnerability, please send an email to:

**q476369545@hotmail.com**

We appreciate your efforts to responsibly disclose the vulnerability. We will aim to acknowledge your report within 48 hours and provide a timeline for the fix.

## Security Best Practices

When using this project, please follow these security guidelines:

- Keep your API keys secure and never commit them to version control
- Use environment variables for sensitive configuration
- Regularly update dependencies to their latest secure versions
- Follow the principle of least privilege when configuring database access

## Scope

This security policy applies to:
- The main application code in this repository
- Dependencies listed in `package.json`
- Build and deployment processes

This policy does NOT apply to:
- Third-party services (LLM providers, database services, etc.)
- User-generated content stored in the application
